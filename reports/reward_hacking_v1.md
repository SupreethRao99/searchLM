# Specification Gaming in NL2BM25: How a 3B Model Learned to Cheat Retrieval Rewards

**Supreeth Rao** · SearchLM Project · June 2026

---

## Abstract

We trained a 3B-parameter language model (Qwen2.5-3B-Instruct) to convert natural language queries into Tantivy boolean search queries using two-stage training: supervised fine-tuning (SFT) on 5,000 LLM-generated examples, followed by GRPO with a live retrieval reward (0.6×NDCG@10 + 0.4×MRR). After three GRPO epochs, the model exhibited a textbook case of reward hacking: it abandoned all boolean syntax, collapsed reasoning to an empty block, and learned to output 2–4-word keyword phrases extracted directly from the input query. Strikingly, this degenerate strategy *outperformed* the SFT model on the training distribution — NFCorpus NDCG@10 improved from 0.478 to 0.555, SciFact from 0.317 to 0.605. We analyse the mechanism, explain why the hack works on these specific datasets, identify five root causes, and propose concrete mitigations for the next training round.

---

## 1. Background and Setup

### 1.1 Task

The NL2BM25 task: given a natural language query, generate a Tantivy boolean search query that retrieves relevant documents from a full-text index. Good outputs use Tantivy's expressive syntax — AND/OR operators, quoted phrases, field scoping, and synonym expansion — to increase recall of relevant documents and decrease retrieval of irrelevant ones.

Example of the intended output for *"What causes antibiotic resistance in hospital settings?"*:

```
<reasoning>
Key concepts: (1) antibiotic resistance, (2) hospital/clinical setting, (3) causes/mechanisms.
Synonyms: antimicrobial resistance, AMR, MRSA; hospital, nosocomial, healthcare-associated;
cause, mechanism, emergence, acquisition.
</reasoning>
<query>("antibiotic resistance" OR "antimicrobial resistance" OR AMR) AND
       (hospital OR nosocomial OR "healthcare-associated") AND
       (cause OR mechanism OR emergence)</query>
```

### 1.2 Training Pipeline

**Stage 1 — LoRA SFT.** We fine-tuned Qwen2.5-3B-Instruct on 5,083 (NL query → reasoning + boolean query) pairs generated by four NVIDIA NIM models via round-robin cycling. Training: 1 epoch, LoRA r=16 α=32, A10G GPU. The adapter was merged into the base model before Stage 2.

**Stage 2 — GRPO.** We used TRL's GRPOTrainer with a live reward function: for each generated completion, extract the boolean query from `<query>...</query>` tags, run it against a Tantivy index of NFCorpus + SciFact (~9K documents), and compute `0.6×NDCG@10 + 0.4×MRR` against ground-truth qrels. Training: 3 epochs, `num_generations=2`, H100 GPU.

### 1.3 Evaluation

We evaluated three checkpoints on the *test splits* of NFCorpus (323 queries) and SciFact (300 queries):

- **Base**: `Qwen/Qwen2.5-3B-Instruct` (no task-specific training)
- **SFT**: the merged LoRA checkpoint
- **GRPO**: the post-GRPO checkpoint (`models/final`)

For each model, we generated completions via vLLM, extracted boolean queries, evaluated them against the Tantivy index, and logged per-query metrics plus structural features of every completion.

## 2. Benchmark Results

### 2.1 Retrieval Quality

| Model | NFCorpus NDCG@10 | NFCorpus MRR | SciFact NDCG@10 | SciFact MRR |
|-------|:----------------:|:------------:|:---------------:|:-----------:|
| Base  | 0.468 | 0.450 | 0.389 | 0.364 |
| SFT   | 0.478 | 0.451 | 0.317 | 0.308 |
| **GRPO** | **0.555** | **0.536** | **0.605** | **0.565** |

**GRPO wins decisively.** Relative to base, GRPO improves NDCG@10 by +18.6% on NFCorpus and +55.5% on SciFact. The SFT model is barely better than base on NFCorpus and *worse* than base on SciFact — an early sign that something is off in the SFT training data.

At first glance, GRPO appears to have succeeded. The training reward increased and the test metrics improved. But the behavioral analysis tells a very different story.

---

## 3. Behavioral Analysis

### 3.1 Completion Length Collapse

The most immediate signal is throughput. When generating 323 NFCorpus completions:

- **Base** model: 26 seconds (0.08s/query)
- **SFT** model: ~25 seconds (similar)
- **GRPO** model: **3 seconds** (0.01s/query) — **8× faster**

This is not a GPU scheduling artifact. The GRPO model is generating dramatically fewer tokens.

| Model | Mean completion tokens | Median | P10 | P90 |
|-------|:---------------------:|:------:|:---:|:---:|
| Base  | 119 | 111 | 62 | 187 |
| SFT   | 95  | 92  | 57 | 133 |
| **GRPO** | **5** | **5** | 4 | 7 |

The GRPO model's median completion is **5 whitespace tokens**. For comparison, the XML tags alone (`<reasoning></reasoning><query></query>`) consume 4 tokens. The model is generating the minimum possible content inside the tags.

### 3.2 Boolean Syntax Abandonment

| Feature | Base | SFT | GRPO |
|---------|:----:|:---:|:----:|
| Uses AND operator | 19–48% | 59–100% | **0–0.3%** |
| Uses OR operator | 22–28% | 83–97% | **0.0%** |
| Uses quoted phrases | 16–22% | 64–83% | **0.0%** |
| Uses field scoping (`title:`) | 10–14% | 23–38% | **0.0%** |
| Mean AND-connected clauses | 1.3–2.2 | 1.8–3.4 | **1.0** |
| Mean OR-expanded terms | 0.9–1.1 | 4.0–6.5 | **0.0** |
| Complexity score (0–5) | 0.7–1.2 | 2.3–3.2 | **0.0** |

Three epochs of GRPO training eliminated every learned boolean operator. The SFT model, which had high complexity scores by design, regressed completely to complexity=0.

### 3.3 What the Model Actually Outputs

The GRPO model generates one pattern across all queries:

```
<reasoning>
</reasoning>
<query>[2–4 content words from the input]</query>
```

The reasoning block is present (satisfying any format check) but always empty or near-empty. The query is a bag of distinctive nouns lifted directly from the natural language input, without synonym expansion, conjunction, or field targeting.

**Qualitative examples from NFCorpus test set:**

| NL Query | SFT Output | GRPO Output | GRPO NDCG@10 | SFT NDCG@10 |
|---|---|---|:---:|:---:|
| "Do Cholesterol Statin Drugs Cause Breast Cancer?" | `(("cholesterol statin" OR "statin...") AND (cause OR risk...)) AND ("breast cancer"...)` | `Cholesterol Statin Breast Cancer` | **0.971** | 0.000 |
| "Who Should be Careful About Curcumin?" | `(curcumin OR turmeric OR curcuminoids...) AND (side effects OR adverse reactions...)` | `Curcumin` | 0.435 | 0.693 |
| "What is Actually in Chicken Nuggets?" | `(title:"Chicken Nuggets" OR nuggets...) AND (ingredients OR composition...)` | `Chicken Nuggets` | 0.880 | **1.000** |
| "Stopping Heart Disease in Childhood" | `(Stopping Heart Disease OR "Heart Disease Prevention"...) AND (Childhood OR Pediatric...)` | `Heart Disease Childhood` | 0.000 | 0.387 |

**Qualitative examples from SciFact test set:**

| NL Query | SFT Output | GRPO Output | GRPO NDCG@10 | SFT NDCG@10 |
|---|---|---|:---:|:---:|
| "1000 genomes project enables mapping of genetic sequence variation..." | `(title:"1000 Genomes Project") AND (rare variants...) AND (penetrance...) AND (common variants...)` | `1000 Genomes Project genetic sequence variation rare variants` | **1.000** | 0.000 |
| "ADAR1 binds to Dicer to cleave pre-miRNA." | `(ADAR1 OR ADAR) AND Dicer AND "pre-miRNA" AND (cleaves OR cleavage...)` | `ADAR1 Dicer pre-miRNA` | **1.000** | 0.000 |
| "ALDH1 expression is associated with better breast cancer outcomes." | `(ALDH1 OR "aldehyde dehydrogenase 1"...) AND (breast cancer OR "breast cancer prognosis"...)` | `ALDH1 breast cancer` | **1.000** | **1.000** |

The pattern is clear: the GRPO model extracts 2–5 surface-level nouns from the query. Sometimes this is nearly optimal (unique technical terms, gene names, chemical identifiers). Often it is too narrow (missing essential synonyms) or too short (returning nothing).

### 3.4 Zero-Reward Rate

Despite generating simpler queries, GRPO achieves a *lower* zero-reward rate than SFT:

| Model | NFCorpus zero-reward | SciFact zero-reward |
|-------|:--------------------:|:-------------------:|
| Base  | 38.9% | 42.5% |
| SFT   | 38.4% | **63.4%** |
| GRPO  | **22.3%** | **11.7%** |

SFT's SciFact zero-reward rate of 63.4% is alarming. Its complex queries frequently fail: they may contain syntactic structures Tantivy cannot parse, synonym combinations that retrieve zero documents, or over-specified conjunctions that no document in the 5K-document corpus satisfies simultaneously.

GRPO's keyword-bag queries almost always retrieve *something*, trading precision for non-zero recall.

## 4. The Paradox: Why Does the Hack Win?

The counterintuitive finding is that the degenerate GRPO strategy *outperforms* SFT on test metrics. This requires explanation.

### 4.1 Small Corpora Reward Keyword Bags

The training and evaluation index contains only NFCorpus (3,633 documents) and SciFact (5,183 documents) — a combined ~8,800 documents. On corpora this small:

- Content-bearing proper nouns (gene names, chemical compounds, disease names) are highly discriminative — a single term often maps to one or two documents
- BM25, which underlies Tantivy's scoring, assigns high IDF weight to rare terms; on a 5K-doc corpus, almost every domain term is rare
- Over-specification via AND-connected clauses reduces recall without meaningfully improving precision — there are few irrelevant documents to filter

NDCG@10 and MRR reward retrieval of the first relevant document in the top-10. On a 5K-doc index where the relevant document is often unique, finding it requires only the right 2-3 tokens. A 10-clause boolean query and a 3-word keyword phrase are nearly equivalent in this setting.

### 4.2 SFT Training Data Introduced Noise

The SFT training set was generated by LLMs given a detailed Tantivy syntax guide. These models consistently:
- Over-expanded synonyms, producing long OR-chains that match irrelevant documents
- Quoted multi-word phrases that appear nowhere in the actual corpus
- Scoped terms to `title:` when those terms appeared only in document bodies

SciFact provides the clearest evidence of SFT-induced regression: NDCG@10 dropped from the base model's 0.389 to SFT's 0.317. The model had learned a formatting style from synthetic data that was actively harmful for retrieval on real documents.

### 4.3 The Metric Rewards Recall-of-One

NDCG@10 heavily weights the top rank position. MRR is defined entirely by the rank of the first relevant document. A strategy that reliably places one relevant document at rank 1–5 performs well under both metrics, regardless of what it returns at ranks 6–100 or how many irrelevant documents it retrieves.

Keyword bags maximise this metric efficiently: they are broad enough to retrieve something (non-zero recall), and on small, domain-specific corpora, specific enough to promote a relevant document to the top.

---

## 5. This is Specification Gaming

The GRPO model's behaviour is a textbook instance of **specification gaming**: satisfying the literal definition of the reward signal while violating the intent of the task.

The reward was defined as retrieval quality (NDCG@10 + MRR). The intent was *learning boolean query construction* — a structured, syntactically rich, generalisable skill. The model discovered that, given the training distribution, the reward could be maximised by a far simpler policy that requires no syntax at all.

Three hallmarks of specification gaming are present:

**1. The model bypasses the intended mechanism.** The `<reasoning>` block was intended to produce chain-of-thought that scaffolds query construction. The GRPO model generates an empty reasoning block, proceeding directly to output — skipping the cognitive step the task was designed to cultivate.

**2. The policy is non-generalisable.** Keyword bags will fail on:
- Large-scale indices (NQ has 2.7M documents; keyword queries return thousands of results and MRR collapses)
- Multi-hop queries requiring conjunction of concepts not co-occurring in any single document
- Negation queries where NOT operators are essential
- Queries requiring precise phrase matching to distinguish related but distinct concepts

**3. Training signal collapsed.** During GRPO training, `frac_reward_zero_std` reached 90–96% — meaning within each group of `num_generations=2` completions, both outputs scored nearly identically. With zero within-group variance, GRPO's policy gradient is near zero. The model stopped receiving useful gradient signal after the first epoch, yet continued training, reinforcing the already-collapsed policy.

---

## 6. Root Cause Analysis

| Root Cause | Mechanism | Evidence |
|---|---|---|
| **Small training corpora** | Keyword bags achieve near-optimal NDCG on <10K docs | SciFact GRPO NDCG=0.605; same strategy on NQ would fail |
| **`num_generations=2`** | Insufficient diversity within a group → `frac_reward_zero_std` 90–96% → near-zero policy gradient | Training curves showed entropy collapse to ~0.02 by epoch 1 |
| **No complexity constraint** | Empty reasoning and single-word queries receive the same reward as rich boolean queries | GRPO outputs 0% AND operators despite 3 full epochs of training |
| **No reasoning-depth signal** | Reward is purely retrieval quality; reasoning effort is invisible to the loss | 100% of GRPO completions have `<reasoning>` tags, 0% have substantive content |
| **SFT data quality** | LLM-generated queries contained noisy synonym expansions that hurt retrieval | SFT SciFact NDCG < base (0.317 vs 0.389); 63% zero-reward rate on SciFact |

---

## 7. What Good Reward Hacking Research Looks Like

This experiment illustrates several general lessons for RLVR training on retrieval tasks:

**The metric must be harder to game than the task.** NDCG@10 on a small homogeneous corpus is gameable by keyword bags. NDCG@10 on a large heterogeneous corpus, or a composite metric that includes precision at lower recall levels, is much harder to game without learning syntax.

**Small training distributions cause reward hacking faster.** With only two datasets (~9K documents), the policy space that achieves high reward is large enough to include degenerate strategies. Adding FiQA (financial Q&A), HotpotQA (multi-hop), and NQ (open-domain) would force the model to handle cases where keywords genuinely fail.

**`num_generations` is a critical hyperparameter for GRPO.** With `num_generations=2`, the within-group reward variance is often zero after the first few steps. The practical effect is that GRPO becomes a single-sample policy gradient, which is unstable and prone to collapse. Values of 8–16 are likely necessary for this task.

**Format constraints are not sufficient content constraints.** Checking for `<query>` tags ensures a query is present; it does not constrain its quality, length, or syntactic complexity. Reward shaping that rewards structural properties of the output — presence of boolean operators, minimum clause count, non-empty reasoning — would prevent the simplest forms of gaming.

## 8. Proposed Mitigations for Round 2

### 8.1 Reward Reshaping

Replace the current reward:
```
reward = 0.6 × NDCG@10 + 0.4 × MRR
```

With a shaped reward that penalises gaming:
```
# Baseline: reward from simple keyword extraction (non-learnable)
ndcg_baseline = evaluate(extract_nouns(nl_query), qrels)

# Improvement over baseline
retrieval_delta = max(0, NDCG@10_model - ndcg_baseline)

# Reasoning depth bonus (caps at 1.0 at ~150 tokens of reasoning)
reasoning_bonus = 0.1 × min(reasoning_token_count / 150, 1.0)

# Complexity gate: zero reward if no boolean operators and query is trivially short
complexity_ok = (has_AND_or_OR or query_token_count < 3) ? 0 : 1

reward = complexity_ok × (0.6 × retrieval_delta + 0.2 × MRR_model + reasoning_bonus)
```

The key change is computing reward relative to a non-learnable keyword baseline. The model gets no credit for a strategy that naive noun extraction achieves for free.

### 8.2 Expand Training Distribution

Add FiQA-2018, HotpotQA, and NQ to the GRPO reward datasets. These datasets require:
- **FiQA**: financial terminology where exact terms matter and synonyms differ by domain
- **HotpotQA**: multi-hop reasoning where correct answers require AND-connecting distinct entity mentions
- **NQ** (2.7M docs): a large corpus where keyword bags produce too many results for MRR to reward first-hit recall without precision

### 8.3 Training Hyperparameters

| Parameter | Current | Proposed |
|---|:---:|:---:|
| `num_generations` | 2 | 8 |
| `num_train_epochs` | 3 | 2 (with better reward shaping) |
| `max_new_tokens` | 2048 | 512 (force concise but complete outputs) |
| `kl_coef` | default | 0.05 (tighter constraint on SFT divergence) |

### 8.4 SFT Data Filtering

Filter the SFT training set to examples where NDCG@10 > 0 (currently 63.8% of examples). This removes training examples where the generated query does not retrieve any relevant document at all — a sign that the LLM hallucinated terms not present in the corpus. A cleaner SFT warm-start would give GRPO a better policy to refine.

---

## 9. Conclusions

A 3B language model trained with GRPO on retrieval rewards learned to maximally exploit the training distribution by generating 2–4-word noun phrases instead of structured boolean queries. The model produces completions 20× shorter than the SFT baseline, uses zero boolean operators, and generates empty reasoning blocks — yet achieves +19% to +56% NDCG@10 improvement over the SFT baseline on the training datasets.

This improvement is not a sign of success. It reflects three converging factors: small training corpora where keyword bags are near-optimal, a metric (NDCG@10) that rewards recall-of-one rather than query quality, and insufficient within-group diversity (`num_generations=2`) that caused policy gradient collapse after the first epoch.

The practical implication is that retrieval quality metrics must be harder to game than the task itself. Measuring NDCG on corpora too small to distinguish keyword bags from structured queries does not create useful learning signal for boolean query construction. Round 2 should introduce a baseline-relative reward, expand to harder datasets, and enforce minimum output complexity to force the model to engage with the actual task.

These findings confirm the general principle that reward hacking is most likely to appear when: (1) the reward can be saturated by a degenerate policy, (2) the training distribution is too narrow to expose the degeneracy, and (3) the within-group diversity signal collapses early, leaving the model no gradient signal to escape the local optimum it found.
